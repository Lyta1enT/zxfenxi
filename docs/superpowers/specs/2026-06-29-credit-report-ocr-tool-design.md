# 征信报告OCR识别与生成工具 - 设计规范

## 1. 项目概述

基于 PyQt5 的桌面 GUI 应用，实现个人征信、企业征信、税务水母报告的 OCR 识别与 Word 报告生成。

**核心流程**: 上传文件 → 预处理 → PaddleOCR 识别 → 规则抽取字段 → 生成 Word 报告

## 2. 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| GUI 框架 | PyQt5 | 成熟的 Python 桌面 GUI 框架 |
| OCR 引擎 | PaddleOCR | 百度开源 OCR，中文识别成熟 |
| PDF 处理 | pdf2image + PyMuPDF | PDF 转图片及文本提取 |
| 文档生成 | python-docx | Word 文档生成 |
| 图片处理 | OpenCV / Pillow | 图片预处理 |

## 3. 架构设计

### 3.1 整体架构

单体桌面应用，所有核心模块在同一进程中运行，耗时操作通过 QThread 在后台执行。

```
┌──────────────────────────────────────────────────┐
│                    MainWindow                      │
│  ┌──────────┐  ┌──────────┐  ┌────────────────┐  │
│  │ 上传组件  │  │ 类型选择  │  │ 进度/日志面板  │  │
│  └────┬─────┘  └──────────┘  └────────────────┘  │
│       │                                            │
│  ┌────▼─────────────────────────────────────────┐  │
│  │          后台工作线程 (Worker QThread)         │  │
│  │  FileHandler → OCREngine → FieldExtractor    │  │
│  │                           → ReportGenerator  │  │
│  └──────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

### 3.2 模块划分

#### 3.2.1 文件处理模块 (FileHandler)
- **职责**: 文件类型判断、格式校验、内容提取
- **支持格式**: PDF, PNG, JPG/JPEG, DOCX, XLSX
- **处理逻辑**:
  - PDF: 先尝试 PyMuPDF 直接提取文本；如提取内容不足（<50字符），转为图片后走 OCR
  - 图片: 直接送入 OCR
  - Word/Excel: 提取已有文本内容（作为补充信息来源）

#### 3.2.2 OCR 引擎模块 (OCREngine)
- **职责**: 封装 PaddleOCR，提供统一识别接口
- **功能**:
  - 图片预处理（灰度化、二值化、去噪、纠偏）
  - 批量识别多张图片
  - 返回结构化结果：文本内容、位置坐标、置信度
  - 按页面组织识别结果

#### 3.2.3 字段抽取模块 (FieldExtractor)
- **职责**: 基于规则模板从 OCR 结果中抽取结构化字段
- **模板管理**: 每种报告类型对应一个抽取规则文件
- **抽取策略**:
  - 关键词精确匹配（如"姓名："、"证件号码："）
  - 关键词+临近位置匹配（如"逾期"附近找数字）
  - 正则表达式匹配（如日期、金额格式）
- **校验规则**:
  - 日期格式校验 (YYYY-MM-DD)
  - 金额格式校验 (数字+单位)
  - 机构名校验（关键词库）

#### 3.2.4 报告生成模块 (ReportGenerator)
- **职责**: 使用 python-docx 生成结构化 Word 报告
- **输出内容**:
  - 摘要表：关键字段汇总
  - 明细表：结构化列表
  - 异常标记：逾期、欠税等异常标注

#### 3.2.5 三种报告模板
| 模板 | 核心字段 |
|------|---------|
| 个人征信 | 姓名、证件号、报告时间、信用卡账户数、贷款账户数、逾期账户数、余额、已结清账户数、异常备注 |
| 企业征信 | 企业名称、统一社会信用代码、报告时间、未结清机构数、余额、短期/中长期借款、担保、公共信息 |
| 水母报告 | 纳税登记状态、是否有滞纳金、欠税金额、近三年开票汇总、税务异常说明 |

### 4.2 交互流程
1. 选择报告类型 → 自动切换对应的字段模板
2. 上传文件（拖拽或选择）
3. 点击「开始处理」→ 启动后台工作线程
4. 实时进度条 + 日志更新
5. 处理完成 → 预览字段结果 → 提示打开输出目录

## 6. 数据流

```
上传文件 → FileHandler 判断类型
         ├── PDF → 尝试提取文本 → 不足则转图片
         ├── 图片 → 直接送 OCR
         ├── DOCX → 提取文本+可选转图片
         └── XLSX → 读取结构化数据
                 ↓
         OCREngine (PaddleOCR)
         ├── 图片预处理
         ├── 文字识别
         └── 返回 [{text, bbox, confidence}, ...]
                 ↓
         FieldExtractor
         ├── 匹配模板规则
         ├── 字段校验
         └── 返回 {field: value, ...}
                 ↓
         ReportGenerator
         ├── 创建 Word 文档
         ├── 写入摘要表 + 明细表 + 异常标记
         └── 保存到 output/
```

## 7. 错误处理

- **文件格式不支持**: 弹窗提示，允许重新选择
- **PDF 解析失败**: 降级到图片转换后 OCR
- **OCR 识别率低**: 日志记录置信度低的文本块，报告中标注为"低置信度"
- **字段未命中**: 标记为"未识别"，报告中留空 + 黄色标注
- **Word 生成失败**: 回滚到临时文件，保留已处理数据

## 8. 边界说明

### 第一版包含
- 支持 PDF、图片、Word、Excel 上传
- 三种报告模板的字段抽取规则（基础版）
- 生成标准 Word 报告
- 进度显示和日志记录
- 异常标记功能

### 第一版不包含
- 批量处理（多文件同时）
- 数据库持久化
- 用户登录/权限
- 字段人工修正界面
- JSON 导出
